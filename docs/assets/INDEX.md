### How driftking Does Audio (Procedural)
- **Engine:** Tone.js `BrownNoise` with speed-dependent pitch
- **Skid:** Tone.js `WhiteNoise` triggered on drift start, released on end
- **Music:** `PolySynth` FM square with simple pattern

**Recommendation:** Use Tone.js for procedural audio. No audio files needed for core gameplay.

See [`docs/research/AUDIO_RESEARCH.md`](../research/AUDIO_RESEARCH.md) for detailed audio research.

## Asset Pipeline

1. **Car models:** Use Kenney GLB files directly (already optimized)
2. **Track pieces:** Use Kenney GLB files, compose tracks in code
3. **Physics colliders:** Extract from GLB triangles (like Racez.io) or use simplified shapes
4. **Audio:** Procedural via Tone.js (no files needed)
5. **Textures:** Use Kenney built-in textures or simple solid colors

### GLB Loading in Three.js

```typescript
import { GLTFLoader } from 'three/addons/loaders/GLTFLoader.js';

const loader = new GLTFLoader();
const gltf = await loader.loadAsync('assets/kenney-car-kit/Models/GLB format/race.glb');
const carModel = gltf.scene;
scene.add(carModel);
```
